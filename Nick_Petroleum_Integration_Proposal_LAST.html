EXECUTIVE SUMMARY

This proposal outlines a comprehensive solution for integrating your cloud-hosted web ERP system with your on-premises Sage Evolution through a professionally-developed API wrapper. Our hybrid architecture seamlessly bridges your Azure-based operations platform with your headquarters-based Sage Evolution, enabling real-time financial data synchronization across 14+ fuel stations.

Understanding Your Current Infrastructure

Your Setup:

  • Sage Evolution: Runs on Windows at your headquarters with local MSSQL database
  • Web ERP Application: Cloud-hosted on Azure (https://ombabyric.azurewebsites.net/)
  • Fuel Stations: 14+ locations accessing the cloud-based web application
  • The Challenge: Your cloud application needs to send financial data to your on-premises Sage Evolution

Our Solution Architecture

We've designed a hybrid architecture that solves the cloud-to-premises integration challenge:

  • API Wrapper: Modern ASP.NET Core REST API that interfaces with Sage Evolution SDK
  • Windows Agent (Package B only): Lightweight service installed on your Sage server that bridges the cloud API with on-premises Sage
  • Sage SDK & Connector: Official Sage components required for any integration (we assist with purchase from Sage, as an authorized Business Partner)

How It Works:

  1. One-Time Setup: We install our lightweight Windows Agent on your Sage Evolution server (15-20 minutes)
  2. Secure Polling: Agent polls our cloud API every 30-60 seconds checking for work (no inbound firewall rules needed)
  3. Local Execution: When your web app sends data, agent receives it, executes Sage SDK operations locally
  4. Cloud Response: Results sent back to cloud API, your web app gets confirmation
  5. Zero Maintenance: We monitor everything, you focus on your business
